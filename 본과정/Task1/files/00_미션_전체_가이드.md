# 🗂️ Codyssey TASK1 미션 완전 가이드 (인덱스)

> 이 시리즈는 *Linux 서버 운영 기초* 미션 전체를, 초보자도 한 단계씩 따라할 수 있게 풀어 쓴 가이드입니다.
> 주제별로 파일을 나눠서, 필요한 부분만 골라 볼 수 있게 만들었어요.

---

## 📋 미션 한 줄 요약

> **"내 Ubuntu 서버 안에 'Agent'라는 앱을 안전하게 띄우고, 1분마다 상태를 자동 점검해서 로그로 남기는 시스템을 구축한다."**

---

## 🎯 왜 이걸 해?

서버 장애가 났을 때 **로그가 없으면 원인 분석은 '감'에 의존**하게 돼요. 실제 회사에서 이런 상황이 발생하면:
- 💸 복구 시간이 수 배로 늘어남
- 🔄 같은 장애가 반복됨
- 😱 다음에도 또 멍하니 화면만 봄

이 미션은 단순한 명령어 암기가 아니에요. **개발 커리어 내내 활용 가능한 안정적인 서버 운영 환경을 직접 구축**하는 연습이에요.

## 📖 핵심 용어 한 번에 정리 (전 파일 공통)

각 가이드에서 자세히 설명되지만, 미리 한눈에 보고 싶다면 여기서 확인하세요.

### 보안 관련

| 용어 | 한 줄 설명 |
|---|---|
| SSH | 원격 접속용 암호화 통신 프로토콜 |
| Port | 컴퓨터의 통신 출입구 번호 (0~65535) |
| 방화벽 | 들어오고 나가는 통신을 규칙대로 허용/차단 |
| UFW | Ubuntu의 표준 방화벽 도구 (Uncomplicated Firewall) |
| root | 모든 권한을 가진 최고 관리자 계정 |

### 계정/권한 관련

| 용어 | 한 줄 설명 |
|---|---|
| User | 시스템 사용자 (사람 한 명에 해당) |
| Group | 여러 사용자를 묶는 단위 (역할별 그룹) |
| sudo | "잠깐만 root 권한 빌려줘"라는 명령어 |
| chmod | 파일/폴더 권한 변경 (read/write/execute) |
| chown | 파일/폴더 소유자 변경 |
| ACL | 기본 권한(rwx)을 넘어 더 세밀한 권한 제어 |
| setfacl/getfacl | ACL 설정/확인 명령어 |

### 시스템/스크립트 관련

| 용어 | 한 줄 설명 |
|---|---|
| 환경변수 | 시스템 어디서나 접근 가능한 *전역 변수* (예: `$HOME`) |
| Shell Script | 명령어들을 자동 실행하게 모아둔 파일 (`.sh`) |
| Shebang | 스크립트 첫 줄의 `#!/bin/bash` (어떤 인터프리터로 실행할지) |
| cron | 정해진 시간에 자동으로 명령어 실행하는 Linux 스케줄러 |
| crontab | cron 작업 목록 (사용자별로 따로 가짐) |
| daemon | 백그라운드에서 항상 돌아가는 프로그램 (예: sshd) |
| process | 실행 중인 프로그램 한 개 (PID로 식별) |
| log | 시스템/앱이 남기는 기록 파일 |

## 🚀 시작하기 전 체크리스트

이 가이드는 다음 환경을 가정해요:

- ✅ macOS (또는 다른 OS)
- ✅ OrbStack (또는 Docker Desktop) 설치 완료
- ✅ Ubuntu 컨테이너 실행 중 (`ubuntu:latest`)
- ✅ 컨테이너 안에서 `root`로 시작

> ⚠️ 만약 위 환경이 안 갖춰졌다면 **`01_SSH_설정_가이드.md`** 의 Step 1~2를 먼저 따라하세요.

## 🎓 이 미션을 다 끝내면 얻는 것

명령어 외운 게 아니라, **이런 사고방식**을 얻어요:

1. **"보안은 한 겹이 아니다"** — 방화벽 + 계정 분리 + 권한 + ACL의 다층 방어
2. **"역할로 나눈다"** — admin/dev/test, 각자 할 일과 권한이 다름
3. **"환경은 변수로 추상화"** — 경로 하드코딩 ❌, 환경변수로 ✅
4. **"자동화는 신뢰의 시작"** — 사람 손 안 가도 점검되어야 진짜 운영
5. **"로그가 있으면 추적 가능"** — 사고 분석은 로그에서 시작
