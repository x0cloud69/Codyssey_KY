## 0. 시작하기 전에 — 환경변수가 뭐야?

### 🤔 한 문장으로

**"시스템 어디서나 접근 가능한 *전역 변수*"** 예요. 

비유로: 회사에 *공용 게시판* 같은 거예요. 누구나 와서 "오늘 본부 회의실은 3층 301호" 같은 정보를 볼 수 있는.

### 🤔 왜 필요해? (하드코딩 vs 환경변수)

#### ❌ 하드코딩 (안 좋은 방식)

코드 안에 경로/포트를 직접 박아 넣는 거:
```python
# Python 코드
LOG_PATH = "/home/agent-admin/agent-app/logs"
PORT = 15034
```

문제:
- 🐛 다른 서버에 옮길 때마다 코드 수정해야 함
- 🐛 개발/운영/테스트 환경마다 다른 값 필요한데 못 함
- 🐛 비밀번호 같은 거 코드에 박으면 → 깃에 올라가서 유출

#### ✅ 환경변수 (좋은 방식)

코드는 *변수만 참조*:
```python
LOG_PATH = os.environ["AGENT_LOG_DIR"]
PORT = int(os.environ["AGENT_PORT"])
```

그리고 실행할 때:
```bash
export AGENT_LOG_DIR=/var/log/agent-app
export AGENT_PORT=15034
python3 agent_app.py
```

장점:
- ✅ 코드 안 건드리고 환경 바꿈
- ✅ 서버마다 다른 값 가능
- ✅ 비밀번호는 환경변수로만 (코드에 안 박음)

### 🤔 이번 미션에서 만들 환경변수

| 변수명 | 값 | 의미 |
|---|---|---|
| `AGENT_HOME` | `/home/agent-admin/agent-app` | 앱 홈 디렉토리 |
| `AGENT_PORT` | `15034` | 앱이 듣는 포트 |
| `AGENT_UPLOAD_DIR` | `$AGENT_HOME/upload_files` | 업로드 폴더 |
| `AGENT_KEY_PATH` | `$AGENT_HOME/api_keys/t_secret.key` | API 키 파일 경로 |
| `AGENT_LOG_DIR` | `/var/log/agent-app` | 로그 디렉토리 |

> 💡 `$AGENT_HOME` 처럼 **다른 변수를 안에 쓸 수 있어요**. 이걸 *변수 확장*이라고 해요. 그래서 `AGENT_HOME` 하나만 바꾸면 나머지가 다 따라옴.

## 1. 핵심 용어 미리 알기

| 용어 | 비유 | 진짜 의미 |
|---|---|---|
| **환경변수** | 공용 게시판 | 시스템 전역 변수 |
| **`export`** | "게시판에 게시" | 변수를 자식 프로세스에도 전달 |
| **`echo $VAR`** | "게시판 내용 보기" | 환경변수 값 출력 |
| **`.bashrc`** | "내 책상 위 메모지" | 사용자 로그인 시 자동 실행되는 파일 |
| **`/etc/environment`** | "회사 전체 공지판" | 시스템 전체에 적용되는 환경변수 |
| **shell** | 명령 받아주는 직원 | bash, sh, zsh 같은 명령 인터프리터 |
| **subshell** | 임시 임원 | 새로 켜진 셸 (부모 환경 일부 상속) |

## 2. 환경변수의 3가지 적용 범위

환경변수를 *어디에* 설정하느냐에 따라 적용 범위가 달라져요.

### 🟢 임시 (현재 셸에서만)

```bash
export AGENT_HOME=/home/agent-admin/agent-app
```

- ✅ 즉시 적용
- ❌ 터미널 닫으면 사라짐
- ✅ 테스트할 때 쓰기 좋음

### 🟡 사용자별 영구 (특정 계정에서만)

`~/.bashrc` 또는 `~/.bash_profile`에 추가:
```bash
echo 'export AGENT_HOME=/home/agent-admin/agent-app' >> ~/.bashrc
```

- ✅ 영구 적용
- ✅ 해당 사용자만 영향
- ❌ 새 셸을 열거나 `source ~/.bashrc` 해야 적용

### 🔴 시스템 전역 (모든 사용자)

`/etc/environment` 또는 `/etc/profile.d/*.sh`에 추가:
```bash
echo 'AGENT_HOME=/home/agent-admin/agent-app' >> /etc/environment
```

- ✅ 모든 사용자에게 적용
- ⚠️ root 권한 필요
- ⚠️ 신중히 (잘못하면 모든 사용자에게 영향)

### 🤔 이번 미션엔 뭘 써?

**`~/.bashrc`** 가 정답이에요 (agent-admin의 홈 디렉토리). 왜냐면:
- agent-admin이 cron으로 monitor.sh 돌릴 거니까 그 사용자 환경에서 작동해야 함
- 시스템 전역까지 갈 필요는 없음

## 3. 실습 단계

### Step 1. 환경변수 추가 (agent-admin의 .bashrc)

#### 🎯 목적
agent-admin 계정에 영구 환경변수 등록.

#### ⌨️ 명령어
```bash
# agent-admin으로 전환
su - agent-admin

# .bashrc 끝에 환경변수 추가
cat >> ~/.bashrc << 'EOF'

# === Agent App Environment Variables ===
export AGENT_HOME=/home/agent-admin/agent-app
export AGENT_PORT=15034
export AGENT_UPLOAD_DIR=$AGENT_HOME/upload_files
export AGENT_KEY_PATH=$AGENT_HOME/api_keys/t_secret.key
export AGENT_LOG_DIR=/var/log/agent-app
EOF

# 변경사항 즉시 적용
source ~/.bashrc
```

#### 🔍 `cat >> ... << 'EOF' ... EOF` 뜯어보기

이 패턴을 **Heredoc(히어독)** 이라고 해요. 여러 줄을 한 번에 파일에 추가하는 기법:

| 부분 | 의미 |
|---|---|
| `cat` | 파일 내용 출력 |
| `>>` | 파일 끝에 *추가* (덮어쓰기 ❌, 추가 ✅) |
| `~/.bashrc` | 대상 파일 |
| `<< 'EOF'` | "다음 EOF가 나올 때까지의 내용을 입력으로 받아" |
| 본문 | 추가할 내용 |
| `EOF` | "여기서 끝" |

> 💡 `'EOF'`처럼 따옴표로 감싸면 **변수 확장 안 됨**. 즉 `$AGENT_HOME`이 그대로 문자로 들어감 (나중에 사용 시점에 확장됨). 따옴표 없으면 *그 자리에서* 확장됨.

#### 🔍 `source ~/.bashrc` 의미
- `source` = "이 파일 내용을 *현재 셸*에서 즉시 실행해줘"
- 동의어: 점(`.`) — `. ~/.bashrc`
- 이걸 안 하면 새 터미널 열어야 환경변수가 적용됨

#### ✅ 확인
```bash
echo "AGENT_HOME=$AGENT_HOME"
echo "AGENT_PORT=$AGENT_PORT"
echo "AGENT_UPLOAD_DIR=$AGENT_UPLOAD_DIR"
echo "AGENT_KEY_PATH=$AGENT_KEY_PATH"
echo "AGENT_LOG_DIR=$AGENT_LOG_DIR"
```

전부 값이 보이면 성공. 빈 줄로 나오면 → `source` 했는지 확인.

#### 💡 한 번에 다 보고 싶다면
```bash
env | grep AGENT_
```

`env`는 모든 환경변수를 보여주는 명령. `grep AGENT_`로 우리 변수만 필터링.

### Step 2. API 키 파일 생성

#### 🎯 목적
앱이 사용할 API 키 파일 만들기.

#### ⌨️ 명령어
```bash
# 키 파일 생성 (한 줄짜리)
echo "agent_api_key_test" > $AGENT_KEY_PATH

# 권한 강하게 (소유자만 읽기/쓰기)
chmod 640 $AGENT_KEY_PATH
```

#### 🔍 뜯어보기
- `echo "..." > 파일` → 출력을 파일에 *덮어쓰기*
- `chmod 640` → 소유자 R/W (6) + 그룹 R (4) + 나머지 X (0) = `rw-r-----`

#### 🤔 왜 권한이 640이야?
API 키는 **민감 정보**예요. 그래서:
- 소유자(agent-admin)는 읽기/쓰기 가능
- 그룹(agent-core)은 *읽기만* 가능 (앱이 읽어야 하니까)
- 나머지는 차단

#### ✅ 확인
```bash
ls -la $AGENT_KEY_PATH
cat $AGENT_KEY_PATH
```

- 권한이 `-rw-r-----`로 보이면 OK
- 내용이 `agent_api_key_test`로 보이면 OK

### Step 4. Agent 앱 코드 배치

#### 🎯 목적
제공받은 Python 앱 파일을 적절한 위치에 배치.

#### 🤔 앱 파일이 어디 있나?
미션에선 "제공 Python 앱"이라고만 돼 있어요. 보통 이런 경로에 두면 깔끔해요:
- `$AGENT_HOME/agent_app.py`

#### ⌨️ 명령어 (앱 파일이 이미 컨테이너에 있다고 가정)
```bash
# agent-admin으로 전환
su - agent-admin

# 앱 파일을 AGENT_HOME에 복사 (원본 위치는 상황에 맞게)
cp /path/to/agent_app.py $AGENT_HOME/

# 실행 권한 부여
chmod 750 $AGENT_HOME/agent_app.py
```

#### 🤔 앱 파일이 없다면?
미션에서 제공해주는 파일이 있을 거예요 (Codyssey 자료실). 일단 *간단한 예시 앱*을 만들어 테스트해볼 수 있어요:

```bash
cat > $AGENT_HOME/agent_app.py << 'EOF'
#!/usr/bin/env python3
"""Minimal Agent App for testing"""
import os
import socket
import time

def boot_step(n, name):
    print(f"[{n}/5] {name}... [OK]")
    time.sleep(0.3)

def main():
    print("=== Agent App Boot Sequence ===")
    boot_step(1, "Loading environment")
    home = os.environ.get("AGENT_HOME", "")
    port = int(os.environ.get("AGENT_PORT", "15034"))
    
    boot_step(2, "Checking API key")
    key_path = os.environ.get("AGENT_KEY_PATH", "")
    if not os.path.exists(key_path):
        print(f"ERROR: key file not found: {key_path}")
        return
    
    boot_step(3, "Preparing upload directory")
    boot_step(4, "Initializing log writer")
    boot_step(5, "Opening port")
    
    sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
    sock.bind(("0.0.0.0", port))
    sock.listen(5)
    
    print("Agent READY")
    print(f"Listening on 0.0.0.0:{port}")
    
    try:
        while True:
            conn, addr = sock.accept()
            conn.send(b"Agent OK\n")
            conn.close()
    except KeyboardInterrupt:
        print("\nShutting down...")
    finally:
        sock.close()

if __name__ == "__main__":
    main()
EOF

chmod 750 $AGENT_HOME/agent_app.py
```

이 코드는 *예시*예요. 실제 제공된 파일이 있으면 그걸 쓰세요.

### Step 5. 앱 실행 (일반 계정으로!)

#### 🎯 목적
드디어 앱 실행 + Boot Sequence 확인.

#### ⌨️ 명령어 (agent-admin으로)
```bash
# agent-admin이어야 함 (root 절대 X)
whoami    # → agent-admin 확인

# 앱 실행
python3 $AGENT_HOME/agent_app.py
```

#### ⚠️ 왜 일반 계정이어야 해?

요구사항: **"일반 계정으로 실행(루트 실행 금지)"**

이유:
- 🛡️ **보안**: root로 앱을 돌리면, 앱이 해킹당했을 때 시스템 전체가 통째로 뚫림
- 🛡️ **최소 권한**: 앱이 필요한 것 이상의 권한을 가지면 안 됨
- 🐛 **실수 방지**: root로 돌리는 앱은 실수로 시스템 파일 망가뜨릴 수 있음

> 💡 이건 *진짜* 중요한 원칙이에요. 운영 환경에서 앱을 root로 띄우는 건 거의 항상 잘못된 일.

#### ✅ 성공 결과
```text
=== Agent App Boot Sequence ===
[1/5] Loading environment... [OK]
[2/5] Checking API key... [OK]
[3/5] Preparing upload directory... [OK]
[4/5] Initializing log writer... [OK]
[5/5] Opening port... [OK]
Agent READY
Listening on 0.0.0.0:15034
```

체크포인트:
- ✅ 5단계 모두 `[OK]`
- ✅ 마지막에 `Agent READY` 출력
- ✅ 포트 15034로 LISTEN 시작

### Step 6. 앱이 진짜 떴는지 *다른 터미널*에서 확인

#### 🎯 목적
앱이 LISTEN 상태인지 검증. 

#### ⌨️ 새 터미널 열기

기존 터미널은 앱 실행 중이라 안 막힘. 컨테이너에 *다른 세션*으로 접속:

```bash
# Mac 터미널에서
docker exec -it ubuntu-ssh bash
```

#### ⌨️ 확인 명령어
```bash
# 프로세스 확인
ps -ef | grep agent_app

# 포트 LISTEN 확인
ss -tlnp | grep 15034
```

#### ✅ 성공 결과
```text
# ps 결과
agent-a+   12345  ... python3 /home/agent-admin/agent-app/agent_app.py

# ss 결과
LISTEN  0  5  0.0.0.0:15034  0.0.0.0:*  users:(("python3",pid=12345,fd=3))
```

핵심:
- ✅ `agent-admin` 사용자로 돌고 있음 (root 아님)
- ✅ `0.0.0.0:15034`에서 LISTEN 중

### Step 7. 앱 종료

#### 🎯 목적
정상 종료 방법.

#### ⌨️ 명령어
앱이 돌고 있는 터미널에서:
```text
Ctrl+C
```

#### 🔍 Ctrl+C는 뭐 하는 거야?
- 실행 중인 프로세스에 **SIGINT** (Interrupt Signal) 보내기
- 프로그램이 "그만!"하고 종료 절차를 실행하도록 유도
- 대부분의 앱은 SIGINT 받으면 자원 정리 후 깔끔하게 종료

#### 🚨 만약 안 죽는다면?
```bash
# 새 터미널에서
pkill -f agent_app.py     # 이름으로 죽이기

# 그래도 안 죽으면 강제
pkill -9 -f agent_app.py  # SIGKILL (절대 종료)
```

`-9`는 강제 종료. 보통은 안 쓰는 게 좋아요 (자원 정리 안 함). 마지막 수단.

## 4. 자주 만나는 에러와 해결법

### ❌ `Permission denied` (앱 실행 시)
**원인**: 파일에 실행 권한 없음 또는 파일 소유자가 다름
**해결**:
```bash
chmod 750 $AGENT_HOME/agent_app.py
chown agent-admin:agent-core $AGENT_HOME/agent_app.py
```

---

### ❌ `Address already in use` (포트 충돌)
**원인**: 15034 포트를 이미 다른 프로세스가 쓰고 있음
**해결**:
```bash
# 누가 쓰는지 확인
ss -tlnp | grep 15034

# 죽이기
pkill -f agent_app.py
```

---

### ❌ `Permission denied` (포트 바인딩 시)
**원인**: 1024 미만 포트는 root만 사용 가능. 15034는 1024 이상이라 일반 사용자도 OK. 만약 권한 에러가 나면 다른 이유.
**해결**: 포트가 1024 미만이면 1024 이상으로 변경

---

### ❌ `FileNotFoundError: api_keys/t_secret.key`
**원인**: 키 파일 안 만듦 또는 경로 오타
**해결**:
```bash
echo "agent_api_key_test" > $AGENT_KEY_PATH
ls -la $AGENT_KEY_PATH
```

---

### ❌ 환경변수가 안 보임 (`echo $AGENT_HOME` → 빈 줄)
**원인**: `.bashrc` 변경 후 `source` 안 함, 또는 새 셸이 아님
**해결**:
```bash
source ~/.bashrc

# 또는 새 셸 시작
bash
```

---

### ❌ `python3: command not found`
**원인**: Python 미설치
**해결**:
```bash
# root로 전환 후
apt install -y python3
```

---

### ❌ 환경변수가 cron에서 안 먹힘 (다음 가이드 예고)
**원인**: cron은 *최소 환경*으로 실행돼서 `.bashrc`를 안 읽음
**해결**: cron 작업 안에서 환경변수 직접 export하거나, `bash -l -c "..."` 사용 (자세한 건 06번 가이드)

## 5. 전체 흐름 한눈에 보기

```text
[ root 권한으로 시작 ]
     │
     │ apt install -y python3 python3-pip       ← Python 준비
     │ su - agent-admin                         ← 일반 계정으로 전환
     │
[ agent-admin 권한 ]
     │
     │ cat >> ~/.bashrc << EOF                  ← 환경변수 등록
     │   export AGENT_HOME=...                  
     │   export AGENT_PORT=15034
     │   export AGENT_UPLOAD_DIR=...
     │   export AGENT_KEY_PATH=...
     │   export AGENT_LOG_DIR=...
     │ EOF
     │ source ~/.bashrc                         ← 즉시 적용
     │
     │ echo "agent_api_key_test" > $AGENT_KEY_PATH    ← 키 파일 생성
     │ chmod 640 $AGENT_KEY_PATH
     │
     │ cp /path/to/agent_app.py $AGENT_HOME/   ← 앱 배치
     │ chmod 750 $AGENT_HOME/agent_app.py
     │
     │ python3 $AGENT_HOME/agent_app.py        ← 실행!
     │   → 5단계 [OK] + "Agent READY"
     │
     │ (다른 터미널에서)
     │ ss -tlnp | grep 15034                   ← LISTEN 확인
     │
     │ (원래 터미널에서)
     │ Ctrl+C                                  ← 종료
     ▼
[ 완료 ]
```

## 📌 마무리: 이번에 진짜 배운 것

1. **"환경변수 = 시스템 공용 게시판"** — 어디서나 접근 가능한 설정
2. **"하드코딩 금지"** — 경로, 포트, 키는 환경변수로
3. **"임시 vs 영구 vs 전역"** — 적용 범위를 의식하고 선택
4. **"`source`로 즉시 적용"** — 새 셸 열 필요 없음
5. **"root로 앱 띄우지 마라"** — 항상 일반 계정으로
6. **"Ctrl+C로 정상 종료"** — `kill -9`는 최후의 수단
7. **"LISTEN 검증 = 진짜 켜졌나 확인"** — `ss -tlnp` 가 친구
