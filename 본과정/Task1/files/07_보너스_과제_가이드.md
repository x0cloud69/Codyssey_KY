# 보너스 1: report.sh로 요약 리포트 자동 생성

## 🎯 목표

`/var/log/agent-app/monitor.log`를 분석해서:
- CPU/MEM/DISK 각각의 **평균(avg), 최대(max), 최소(min)** 계산
- 분석한 **샘플 수** 출력
- (선택) 시작/종료 시간 입력받아 *해당 구간만* 분석

## 🤔 한 줄 요약

**"누적된 로그를 *통계로 요약*해주는 스크립트"**.

monitor.log는 매분 1줄씩 쌓이는데, 시간이 지나면 1만 줄, 10만 줄이 돼요. 사람이 일일이 못 봐요. → 요약 리포트가 필요.

---

## 핵심 도구: awk

이번 스크립트의 주인공은 **awk**예요.

### awk가 뭐야?

**"줄 단위로 데이터를 처리하는 *작은 프로그래밍 언어*"** 입니다.

비유: 엑셀의 SUMIF, AVG 같은 기능을 *명령줄에서* 쓰는 도구.

### 기본 사용 예

```bash
echo "apple 100" | awk '{print $1, $2}'    # apple 100
echo "apple 100" | awk '{print $2 + 50}'   # 150
```

- `$1`, `$2` = 1번째, 2번째 단어
- `$0` = 전체 줄
- `{...}` = 각 줄마다 실행할 코드

---

## report.sh 전체 코드

```bash
#!/bin/bash
###############################################################################
# report.sh - Agent monitor.log Analysis Report
#
# 사용법:
#   ./report.sh                               # 전체 로그 분석
#   ./report.sh "2026-05-18 10:00:00" "2026-05-18 12:00:00"   # 구간 분석
###############################################################################

LOG_FILE="/var/log/agent-app/monitor.log"

# ─────────────────────────────────────────────────────────────
# 1. 파일 존재 확인
# ─────────────────────────────────────────────────────────────
if [ ! -f "$LOG_FILE" ]; then
    echo "ERROR: Log file not found: $LOG_FILE"
    exit 1
fi

# ─────────────────────────────────────────────────────────────
# 2. 시작/종료 시간 (선택적 인자)
# ─────────────────────────────────────────────────────────────
START_TIME="${1:-}"   # 첫 번째 인자, 없으면 빈 값
END_TIME="${2:-}"     # 두 번째 인자, 없으면 빈 값

# 분석 대상 결정
if [ -n "$START_TIME" ] && [ -n "$END_TIME" ]; then
    echo "=== Analysis Range: $START_TIME ~ $END_TIME ==="
    # awk로 구간 필터링
    DATA=$(awk -v start="$START_TIME" -v end="$END_TIME" '
        {
            # 첫 두 단어가 날짜+시간 (대괄호 제거 후)
            ts = $1 " " $2
            gsub(/\[|\]/, "", ts)
            if (ts >= start && ts <= end) print
        }
    ' "$LOG_FILE")
else
    echo "=== Analysis: All Logs ==="
    DATA=$(cat "$LOG_FILE")
fi

# ─────────────────────────────────────────────────────────────
# 3. 데이터가 있는지 확인
# ─────────────────────────────────────────────────────────────
# WARNING 줄은 제외하고, 메인 라인만
DATA=$(echo "$DATA" | grep -E "PID:.*CPU:.*MEM:.*DISK_USED:")
SAMPLE_COUNT=$(echo "$DATA" | grep -c .)

if [ "$SAMPLE_COUNT" -eq 0 ]; then
    echo "No data found in the specified range."
    exit 0
fi

# ─────────────────────────────────────────────────────────────
# 4. 통계 계산
# ─────────────────────────────────────────────────────────────
echo ""
echo "Samples: $SAMPLE_COUNT"
echo ""
printf "%-8s %10s %10s %10s\n" "METRIC" "AVG" "MAX" "MIN"
printf "%-8s %10s %10s %10s\n" "------" "------" "------" "------"

# awk로 한 번에 처리 (CPU, MEM, DISK 각각)
echo "$DATA" | awk '
    {
        # 정규식으로 숫자 추출
        match($0, /CPU:([0-9]+)/, c)
        match($0, /MEM:([0-9]+)/, m)
        match($0, /DISK_USED:([0-9]+)/, d)
        
        cpu = c[1] + 0
        mem = m[1] + 0
        disk = d[1] + 0
        
        cpu_sum += cpu; mem_sum += mem; disk_sum += disk
        n++
        
        # max 초기화 및 업데이트
        if (n == 1) {
            cpu_max = cpu_min = cpu
            mem_max = mem_min = mem
            disk_max = disk_min = disk
        } else {
            if (cpu > cpu_max) cpu_max = cpu
            if (cpu < cpu_min) cpu_min = cpu
            if (mem > mem_max) mem_max = mem
            if (mem < mem_min) mem_min = mem
            if (disk > disk_max) disk_max = disk
            if (disk < disk_min) disk_min = disk
        }
    }
    END {
        if (n > 0) {
            printf "%-8s %9.1f%% %9d%% %9d%%\n", "CPU",  cpu_sum/n,  cpu_max,  cpu_min
            printf "%-8s %9.1f%% %9d%% %9d%%\n", "MEM",  mem_sum/n,  mem_max,  mem_min
            printf "%-8s %9.1f%% %9d%% %9d%%\n", "DISK", disk_sum/n, disk_max, disk_min
        }
    }
'

exit 0
```

## 코드 한 블록씩 뜯어보기

### 🔍 1. 인자 받기

```bash
START_TIME="${1:-}"
END_TIME="${2:-}"
```

- `$1`, `$2` = 스크립트에 전달된 첫 번째, 두 번째 인자
- `${1:-}` = "$1이 없으면 빈 값으로 처리"
- 사용 예:
  ```bash
  ./report.sh                                 # 인자 없음 → 전체
  ./report.sh "2026-05-18 10:00:00" "12:00:00"   # 인자 둘 → 구간
  ```

### 🔍 2. 구간 필터링

```bash
DATA=$(awk -v start="$START_TIME" -v end="$END_TIME" '
    {
        ts = $1 " " $2
        gsub(/\[|\]/, "", ts)
        if (ts >= start && ts <= end) print
    }
' "$LOG_FILE")
```

- `-v start="..."` → awk 안으로 *외부 변수 전달*
- `$1 " " $2` → "[2026-05-18" + " " + "10:00:00]"
- `gsub(/\[|\]/, "", ts)` → 대괄호(`[`, `]`) 제거
- 결과: `2026-05-18 10:00:00` 형태로 비교 가능

### 🔍 3. 정규식으로 숫자 추출

```bash
match($0, /CPU:([0-9]+)/, c)
cpu = c[1] + 0
```

- `match(문자열, 패턴, 결과배열)` → 패턴 매칭
- `c[1]` → 첫 번째 캡처 그룹 (괄호 안)
- `+ 0` → 문자열을 *숫자로 변환* (awk 트릭)

### 🔍 4. 통계 누적

```bash
cpu_sum += cpu
n++

if (cpu > cpu_max) cpu_max = cpu
```

- 매 줄마다 합계에 더하고, 개수 증가
- 최대/최소 갱신

### 🔍 5. 최종 출력 (END 블록)

```bash
END {
    if (n > 0) {
        printf "%-8s %9.1f%% %9d%% %9d%%\n", "CPU", cpu_sum/n, cpu_max, cpu_min
        ...
    }
}
```

- `END {}` → 모든 줄 처리 *후* 한 번 실행
- `printf` → 형식화된 출력
- `%-8s` = 8자리 왼쪽 정렬 문자열
- `%9.1f` = 9자리, 소수점 1자리 실수
- `%9d` = 9자리 정수

# 보너스 2: 시간 기반 로그 보존 정책

## 🎯 목표

시간이 지난 로그를 **자동으로** 처리:

| 경과 시간 | 처리 |
|---|---|
| 7일 경과 | **압축** (.gz) |
| 그 후 | **아카이브 폴더로 이동** |
| 30일 경과 | **삭제** |

## 🤔 왜 이런 정책이 필요해?

### ❌ 정책 없으면

- 디스크가 점점 차서 결국 *서버 다운*
- 로그가 너무 많아서 *분석 도구가 느려짐*
- 옛날 로그를 *영영 못 지움*

### ✅ 정책 있으면

- 디스크 공간 *자동 관리*
- 최근 로그는 *바로 검색 가능*
- 옛날 로그는 *압축돼서 작게 저장*
- 너무 오래된 건 *자동 삭제*

이런 라이프사이클을 **로그 로테이션(Log Rotation)** 이라고 해요.

---

## 핵심 도구들

### `find` — 파일 찾기 마법사

```bash
# 7일보다 오래된 .log 파일 찾기
find /var/log/agent-app -name "*.log" -mtime +7

# -mtime +7 = 수정된 지 7일 *이상* 지난
# -mtime -7 = 수정된 지 7일 *이내*
```

### `gzip` / `gunzip` — 압축/해제

```bash
gzip file.log         # file.log → file.log.gz
gunzip file.log.gz    # file.log.gz → file.log
```

### `mv` — 이동

```bash
mv file.gz /var/log/archive/
```

---

## 로그 보존 스크립트 전체 코드

```bash
#!/bin/bash
###############################################################################
# log_retention.sh - Agent Log Retention Policy
# 
# - 7일 이상 된 .log 파일 → 압축 후 아카이브로 이동
# - 30일 이상 된 .gz 파일 → 삭제
###############################################################################

# ─────────────────────────────────────────────────────────────
# 설정
# ─────────────────────────────────────────────────────────────
LOG_DIR="/var/log/agent-app"
ARCHIVE_DIR="/var/log/monitor/agent-app/archive"
COMPRESS_DAYS=7
DELETE_DAYS=30
TIMESTAMP=$(date '+%Y-%m-%d %H:%M:%S')

# ─────────────────────────────────────────────────────────────
# 안전장치: 디렉토리 존재 확인
# ─────────────────────────────────────────────────────────────
if [ ! -d "$LOG_DIR" ]; then
    echo "[$TIMESTAMP] [WARNING] Log directory not found: $LOG_DIR"
    exit 0
fi

# 아카이브 디렉토리는 없으면 자동 생성
mkdir -p "$ARCHIVE_DIR" 2>/dev/null
if [ ! -w "$ARCHIVE_DIR" ]; then
    echo "[$TIMESTAMP] [ERROR] Archive directory not writable: $ARCHIVE_DIR"
    exit 1
fi

# ─────────────────────────────────────────────────────────────
# 1. 7일 이상 된 .log 파일 압축
# ─────────────────────────────────────────────────────────────
echo "[$TIMESTAMP] Looking for *.log files older than $COMPRESS_DAYS days..."

COMPRESS_COUNT=0
while IFS= read -r -d '' file; do
    # monitor.log (현재 active 로그)는 제외
    if [ "$(basename "$file")" = "monitor.log" ]; then
        continue
    fi
    
    if gzip "$file" 2>/dev/null; then
        # 압축 성공 → 아카이브로 이동
        mv "${file}.gz" "$ARCHIVE_DIR/" 2>/dev/null
        COMPRESS_COUNT=$((COMPRESS_COUNT + 1))
        echo "  - Compressed and moved: $(basename "$file").gz"
    else
        echo "  - [WARNING] Failed to compress: $file"
    fi
done < <(find "$LOG_DIR" -maxdepth 1 -type f -name "*.log" -mtime +$COMPRESS_DAYS -print0 2>/dev/null)

echo "[$TIMESTAMP] Compressed and archived $COMPRESS_COUNT file(s)."

# ─────────────────────────────────────────────────────────────
# 2. 30일 이상 된 아카이브(.gz) 삭제
# ─────────────────────────────────────────────────────────────
echo "[$TIMESTAMP] Looking for *.gz files older than $DELETE_DAYS days in archive..."

DELETE_COUNT=0
while IFS= read -r -d '' file; do
    if rm -f "$file"; then
        DELETE_COUNT=$((DELETE_COUNT + 1))
        echo "  - Deleted: $(basename "$file")"
    fi
done < <(find "$ARCHIVE_DIR" -maxdepth 1 -type f -name "*.gz" -mtime +$DELETE_DAYS -print0 2>/dev/null)

echo "[$TIMESTAMP] Deleted $DELETE_COUNT old archive(s)."

exit 0
```

## 코드 뜯어보기

### 🔍 `find ... -print0` + `read -d ''` 패턴

```bash
while IFS= read -r -d '' file; do
    ...
done < <(find ... -print0)
```

이 복잡한 패턴이 왜 필요할까요?

**이유**: 파일 이름에 *공백이 있을 수 있어요*. 단순히 `for file in $(find ...)` 하면 공백 있는 이름이 *여러 단어로 쪼개짐*.

해결책:
- `-print0` → 파일 이름들을 *null 문자*로 구분
- `read -d ''` → null 문자로 구분된 입력 읽기
- 결과: 공백 있는 이름도 안전하게 처리

> 💡 운영 환경에서 *이 패턴이 표준*이에요. 외워두면 평생 도움.

### 🔍 `mkdir -p ... 2>/dev/null`

- `-p` → 중간 디렉토리도 생성, 이미 있어도 에러 안 남
- `2>/dev/null` → 만약 에러가 나도 화면에 안 보이게

### 🔍 `[ -w 디렉토리 ]`

- `-w` = writable (쓰기 가능한가?)
- 권한 미리 체크 → 진행 중 에러 방지

### 🔍 `find -maxdepth 1`

- `-maxdepth 1` → 해당 디렉토리만, 하위 폴더 안 들어감
- 안전장치 (의도치 않은 하위 폴더 처리 방지)

### 🔍 `find -mtime +N`

- `+N` = N일 *이상*
- `-N` = N일 *이내*
- `N` = 정확히 N일

## cron으로 자동 실행

이 스크립트는 매분 돌릴 필요 없어요. **하루에 한 번**이면 충분.

```bash
# agent-admin의 crontab 편집
crontab -u agent-admin -e

# 추가 (매일 새벽 3시)
0 3 * * * /bin/bash -lc '/home/agent-admin/agent-app/bin/log_retention.sh' >> /var/log/agent-app/retention.log 2>&1
```

### 🤔 왜 새벽 3시?

운영 트래픽이 가장 적은 시간이라 *I/O 부하*에 영향 적음. 일반적으로 새벽 시간대 (2~5시)가 정비 작업의 표준.

## 시스템 도구로 더 간단히: logrotate

위 스크립트를 직접 짜는 대신, Linux 표준 도구 `logrotate`를 쓸 수도 있어요.

### 설치
```bash
apt install -y logrotate
```

### 설정 파일 작성
```bash
cat > /etc/logrotate.d/agent-app << 'EOF'
/var/log/agent-app/*.log {
    daily               # 매일 검사
    rotate 7            # 7개까지 보관
    compress            # gzip 압축
    delaycompress       # 가장 최근 건 압축 안 함
    missingok           # 파일 없어도 에러 X
    notifempty          # 빈 파일은 처리 X
    maxage 30           # 30일 지난 건 삭제
    create 770 agent-admin agent-core   # 새 파일 권한
}
EOF
```

### 테스트
```bash
# 강제 실행 (테스트)
logrotate -f /etc/logrotate.d/agent-app

# 평소엔 cron으로 자동 (보통 /etc/cron.daily에 등록돼 있음)
```

### 🤔 직접 스크립트 vs logrotate

| 방식 | 장점 | 단점 |
|---|---|---|
| **직접 스크립트** | 완전 제어, 학습 효과 | 만들기 귀찮음 |
| **logrotate** | 짧고 안정적 | 설정 문법 따로 배워야 |

**실무**: logrotate를 더 많이 써요. 이번 미션은 *학습 목적*이니 직접 스크립트 짜보는 게 좋아요.

## 자주 만나는 에러와 해결법

### ❌ `gzip: command not found`
**해결**:
```bash
apt install -y gzip
```

---

### ❌ `find: missing argument to '-mtime'`
**원인**: 공백 누락 또는 인자 순서
**해결**: 정확히 `-mtime +7` 형식 (공백 1개)

---

### ❌ 아카이브 디렉토리 권한 에러
**원인**: cron 실행자가 그 경로에 쓰기 권한 없음
**해결**:
```bash
mkdir -p /var/log/monitor/agent-app/archive
chown agent-admin:agent-core /var/log/monitor/agent-app/archive
chmod 770 /var/log/monitor/agent-app/archive
```

---

### ❌ logrotate가 동작 안 함
**확인**:
```bash
# 강제 실행해보기
logrotate -fv /etc/logrotate.d/agent-app

# 로그 확인
cat /var/lib/logrotate/status | grep agent-app
```

---

### ❌ report.sh에서 awk 결과 이상함
**원인**: `match()` 함수의 *세 번째 인자(배열)* 는 **GNU awk(gawk) 전용** 기능
**해결**: Ubuntu는 보통 gawk가 기본. 만약 mawk만 있다면:
```bash
apt install -y gawk
```

## 전체 흐름 한눈에 보기

```text
┌─────────────────────────────────────────────────────┐
│  보너스 1: report.sh                                │
```
